authorize

authorize: ((conversationId, clientId) => Promise<boolean>)

This callback is called by CoreCrypto to know whether a given clientId is authorized to "write" in the given conversationId. Think of it as a "isAdmin" callback conceptually

This callback exists because there are many business cases where CoreCrypto doesn't have enough knowledge (such as what can exist on a backend) to inform the decision

clientIsExistingGroupUser

clientIsExistingGroupUser: ((conversationId, clientId, existingClients, parent_conversation_clients?) => Promise<boolean>)

Callback to ensure that the given clientId belongs to one of the provided existingClients This basically allows to defer the client ID parsing logic to the caller - because CoreCrypto is oblivious to such things

userAuthorize

userAuthorize: ((conversationId, externalClientId, existingClients) => Promise<boolean>)

A mix between authorize and clientIsExistingGroupUser. We currently use this callback to verify external commits to join a group ; in such case, the client has to:

  • first, belong to a user which is already in the MLS group (similar to clientIsExistingGroupUser)
  • then, this user should be authorized to "write" in the given conversation (similar to authorize)
