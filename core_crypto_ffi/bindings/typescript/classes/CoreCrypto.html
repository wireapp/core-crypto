joinByExternalCommit

  • joinByExternalCommit(groupInfo, credentialType, configuration?): Promise<ConversationInitBundle>

  • Allows to create an external commit to "apply" to join a group through its GroupInfo.

    If the Delivery Service accepts the external commit, you have to CoreCrypto.mergePendingGroupFromExternalCommit in order to get back a functional MLS group. On the opposite, if it rejects it, you can either retry by just calling again CoreCrypto.joinByExternalCommit, no need to CoreCrypto.clearPendingGroupFromExternalCommit. If you want to abort the operation (too many retries or the user decided to abort), you can use CoreCrypto.clearPendingGroupFromExternalCommit in order not to bloat the user's storage but nothing bad can happen if you forget to except some storage space wasted.

    Parameters

    • groupInfo: Uint8Array

      a TLS encoded GroupInfo fetched from the Delivery Service

    • credentialType: CredentialType

      kind of Credential to use for joining this group. If CredentialType.Basic is chosen and no Credential has been created yet for it, a new one will be generated.

    • configuration: CustomConfiguration = {}

      configuration of the MLS group When CredentialType.X509 is chosen, it fails when no Credential has been created for the given Ciphersuite.

    Returns Promise<ConversationInitBundle>

    see ConversationInitBundle

    • Defined in CoreCrypto.ts:1720
